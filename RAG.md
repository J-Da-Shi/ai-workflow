 一、RAG 是什么
                                                                                                                                                                                          
  Retrieval-Augmented Generation（检索增强生成）        
                                                                                                                                                                                          
  核心思想很简单：AI 大模型知识有限（训练数据截止到某个时间点，也不知道你公司的内部文档），所以在提问之前，先帮它搜一遍相关资料，塞进 prompt 里，让它"开卷考试"。                         
                                                                                                                                                                                          
  普通 AI：闭卷考试（只靠训练时学到的知识）                                                                                                                                               
  RAG AI：开卷考试（先翻参考资料，再回答问题）                                                                                                                                            
                                                                                                                                                                                          
  为什么不直接把所有文档塞给 AI？                                                                                                                                                         
                                                                                                                                                                                          
  - 大模型有 上下文窗口限制（GPT-4 128K ≈ 10万字，DeepSeek 64K）                                                                                                                          
  - 塞太多内容 → 费用高、速度慢、AI 注意力分散反而效果差（"大海捞针"问题）
  - RAG 只挑最相关的几段塞进去 → 精准、快、便宜                                                                                                                                           
                                                                                                                                                                                          
  ---                                                                                                                                                                                     
  二、核心概念                                                                                                                                                                            
                                                                                                                                                                                          
  1. Embedding（向量嵌入）                              
                                                                                                                                                                                          
  把文本变成一串数字（向量），语义相近的文本，向量也相近。                                                                                                                                
                                                                                                                                                                                          
  "JavaScript 闭包的用法"  → [0.12, -0.34, 0.56, ..., 0.78]                                                                                                                               
  "JS 如何使用闭包"        → [0.11, -0.33, 0.55, ..., 0.77]  ← 很接近！                                                                                                                   
  "今天吃什么"             → [0.89, 0.23, -0.67, ..., 0.12]  ← 很远                                                                                                                       
                                                                                                                                                                                          
  - 一般是 1024~3072 维的浮点数数组                                                                                                                                                       
  - 由专门的 Embedding 模型生成（不是 ChatGPT，是另一个小模型）                                                                                                                           
  - 一次调用可以批量处理多段文本                                                                                                                                                          
                                                                                                                                                                                          
  2. 向量相似度                                                                                                                                                                           
                                                                                                                                                                                          
  怎么判断两个向量"像不像"？主要两种距离计算：                                                                                                                                            
                                                        
  ┌──────────────┬──────────────────────┬────────────────────────┬────────────────────┐                                                                                                   
  │     方法     │       公式含义       │          值域          │      常用场景      │
  ├──────────────┼──────────────────────┼────────────────────────┼────────────────────┤                                                                                                   
  │ 余弦相似度   │ 两个向量夹角的余弦值 │ -1 ~ 1（1 = 完全相同） │ 文本检索（最常用） │
  ├──────────────┼──────────────────────┼────────────────────────┼────────────────────┤                                                                                                   
  │ 欧几里得距离 │ 两点之间直线距离     │ 0 ~ ∞（0 = 完全相同）  │ 图像检索           │                                                                                                   
  └──────────────┴──────────────────────┴────────────────────────┴────────────────────┘                                                                                                   
                                                                                                                                                                                          
  你可以理解为：余弦相似度看方向（内容主题是否一致），欧氏距离看绝对位置。文本场景用余弦就够了。                                                                                          
                                                        
  3. 向量数据库                                                                                                                                                                           
                                                        
  专门存储向量、做高速相似搜索的数据库。                                                                                                                                                  
                                                        
  MySQL 这类传统数据库做的是精确匹配（WHERE name = 'xxx'），向量数据库做的是模糊语义搜索（找最相似的 top K 条）。                                                                         
                                                        
  底层用的索引算法叫 HNSW（Hierarchical Navigable Small World）：                                                                                                                         
  - 把所有向量组织成一个多层图结构                      
  - 搜索时从顶层粗糙定位，逐层精细化                                                                                                                                                      
  - 不需要跟每条数据都比较，百万级数据也能毫秒级返回    
  - 你不需要理解算法细节，知道"它很快"就行                                                                                                                                                
                                                                                                                                                                                          
  4. Chunk（切片）                                                                                                                                                                        
                                                                                                                                                                                          
  文档太长，不能整篇 Embedding，要切成小段：                                                                                                                                              
                                                                                                                                                                                          
  ┌──────────────────────────────────────┐                                                                                                                                                
  │         一篇 20 页的 PDF 文档          │                                                                                                                                              
  └──────────────────────────────────────┘                                                                                                                                                
                   ↓ 切片                                                                                                                                                                 
  ┌─────────┐ ┌─────────┐ ┌─────────┐                                                                                                                                                     
  │ chunk 1 │ │ chunk 2 │ │ chunk 3 │ ...                                                                                                                                                 
  │ 800字符  │ │ 800字符  │ │ 800字符  │                                                                                                                                                  
  └─────────┘ └─────────┘ └─────────┘                                                                                                                                                     
      ↓ Embedding     ↓           ↓                                                                                                                                                       
   [0.12, ...]    [0.34, ...]  [0.56, ...]                                                                                                                                                
      ↓               ↓           ↓                                                                                                                                                       
           存入向量数据库                                                                                                                                                                 
                                                                                                                                                                                          
  每个 chunk 单独 Embedding，单独存储，搜索时独立匹配。                                                                                                                                   
                                                                                                                                                                                          
  ---                                                                                                                                                                                     
  三、完整 RAG 流程                                     
                                                                                                                                                                                          
  分两个阶段：
                                                                                                                                                                                          
  阶段一：索引（上传文档时，一次性）                                                                                                                                                      
                                                                                                                                                                                          
  1. 文档加载                                                                                                                                                                             
     PDF / Word / Markdown / TXT → 提取纯文本                                                                                                                                             
     ↓                                                                                                                                                                                    
  2. 文本清洗                                                                                                                                                                             
     去掉页眉页脚、多余空行、特殊字符                                                                                                                                                     
     ↓                                                                                                                                                                                    
  3. 切片（Chunking）                                                                                                                                                                     
     按固定长度或语义边界切成 N 个片段                                                                                                                                                    
     每个片段记录 metadata（来源文档、页码、标题等）                                                                                                                                      
     ↓                                                                                                                                                                                    
  4. 向量化（Embedding）                                                                                                                                                                  
     批量调 Embedding API，每个片段 → 一个向量                                                                                                                                            
     ↓                                                                                                                                                                                    
  5. 存入向量数据库                                                                                                                                                                       
     向量 + 原始文本 + metadata → 一条记录                                                                                                                                                
                                                                                                                                                                                          
  阶段二：检索 + 生成（节点执行时，每次）                                                                                                                                                 
                                                                                                                                                                                          
  1. 用户查询（节点的 input 或 prompt）                                                                                                                                                   
     "分析用户登录模块的安全需求"                                                                                                                                                         
     ↓                                                                                                                                                                                    
  2. 查询向量化                                                                                                                                                                           
     调 Embedding API → 得到查询向量                                                                                                                                                      
     ↓                                                                                                                                                                                    
  3. 向量检索                                                                                                                                                                             
     在向量数据库中搜最相似的 top K 个片段                                                                                                                                                
     返回：[{content: "...", score: 0.92}, {content: "...", score: 0.87}, ...]                                                                                                            
     ↓                                                                                                                                                                                    
  4. 构建增强 Prompt                                                                                                                                                                      
     system: "你是一个需求分析师..."                                                                                                                                                      
     user: "以下是相关参考资料：                                                                                                                                                          
           [片段1: 登录安全规范第3章...]                                                                                                                                                  
           [片段2: OAuth2.0 接入标准...]                                                                                                                                                  
           [片段3: 密码策略要求...]                                                                                                                                                       
                                                                                                                                                                                          
           请基于以上资料，分析用户登录模块的安全需求。"                                                                                                                                  
     ↓                                                                                                                                                                                    
  5. 调用 AI 生成                                                                                                                                                                         
     AI 基于参考资料 + 自身知识 → 输出结果                                                                                                                                                
                                                                                                                                                                                          
  ---                                                                                                                                                                                     
  四、切片策略详解                                                                                                                                                                        
                                                                                                                                                                                          
  切片是 RAG 效果的最关键因素。切不好，检索就不准，AI 就答不好。
                                                                                                                                                                                          
  常见策略                                                                                                                                                                                
                                                                                                                                                                                          
  1. 固定长度切片（最简单）                                                                                                                                                               
                                                        
  参数：chunkSize = 800, chunkOverlap = 200                                                                                                                                               
                                                                                                                                                                                          
  原文: [AAAAAAAAAA BBBBBBBBBB CCCCCCCCCC DDDDDDDDDD]                                                                                                                                     
                                                                                                                                                                                          
  chunk1: [AAAAAAAAAA BB]        ← 800字符                                                                                                                                                
  chunk2:            [BB CCCCCCCCCC]  ← 重叠200字符     
  chunk3:                    [CC DDDDDDDDDD]                                                                                                                                              
                                                        
  重叠（overlap）很重要 — 如果一句话刚好被切断，重叠保证上下文连续性。                                                                                                                    
                                                        
  适用：通用文本、博客文章、聊天记录                                                                                                                                                      
                                                        
  2. 递归字符切片（推荐）                                                                                                                                                                 
                                                        
  LangChain 的 RecursiveCharacterTextSplitter，按优先级尝试在不同分隔符处切：                                                                                                             
                                                        
  尝试顺序："\n\n"(双换行) → "\n"(单换行) → " "(空格) → ""(字符)                                                                                                                          
                                                                                                                                                                                          
  优先在段落边界切 → 切不动就在换行处切 → 还切不动就在空格处切                                                                                                                            
                                                                                                                                                                                          
  效果比固定长度好，因为尽量保持语义完整性。                                                                                                                                              
                                                        
  3. 按文档结构切                                                                                                                                                                         
                                                        
  识别 Markdown 标题层级、HTML 标签来切：                                                                                                                                                 
                                                        
  # 第一章 用户认证          ← 切片边界                                                                                                                                                   
  ## 1.1 登录流程            ← 切片边界                                                                                                                                                   
  登录流程包含以下步骤...                                                                                                                                                                 
  ## 1.2 注册流程            ← 切片边界                                                                                                                                                   
  注册需要验证手机号...                                                                                                                                                                   
                                                                                                                                                                                          
  适用：有清晰标题结构的技术文档                                                                                                                                                          
                                                                                                                                                                                          
  4. 语义切片（最高级）                                                                                                                                                                   
                                                        
  用 Embedding 模型检测语义变化点：                                                                                                                                                       
                                                        
  句子1: "Docker 是一个容器化平台"     ← 相似度 0.95                                                                                                                                      
  句子2: "它可以打包应用和依赖"        ← 相似度 0.93                                                                                                                                      
  句子3: "容器可以在任何环境运行"       ← 相似度 0.91                                                                                                                                     
  句子4: "K8s 是容器编排工具"          ← 相似度 0.72 ← 这里切！话题变了                                                                                                                   
  句子5: "它管理多个容器的部署"                                                                                                                                                           
                                                                                                                                                                                          
  效果最好，但每句话都要调 Embedding API，成本高、速度慢。                                                                                                                                
                                                                                                                                                                                          
  chunk 大小怎么选？                                                                                                                                                                      
                                                        
  ┌─────────────────────┬──────────────────────┬──────────────────────────────┐                                                                                                           
  │     chunk 大小      │         优点         │             缺点             │
  ├─────────────────────┼──────────────────────┼──────────────────────────────┤                                                                                                           
  │ 小（200-500字符）   │ 检索精准，匹配粒度细 │ 上下文碎片化，AI 可能看不懂  │
  ├─────────────────────┼──────────────────────┼──────────────────────────────┤                                                                                                           
  │ 中（500-1000字符）  │ 平衡精准度和上下文   │ 通用推荐                     │                                                                                                           
  ├─────────────────────┼──────────────────────┼──────────────────────────────┤                                                                                                           
  │ 大（1000-2000字符） │ 上下文完整           │ 检索不够精准，消耗 tokens 多 │                                                                                                           
  └─────────────────────┴──────────────────────┴──────────────────────────────┘                                                                                                           
                                                        
  建议起步用 800 字符、200 重叠，后面根据效果调整。                                                                                                                                       
                                                        
  ---                                                                                                                                                                                     
  五、检索优化                                          
                                                                                                                                                                                          
  基础 RAG 做完后，效果可能不够好。以下是常见优化手段，按实施难度排序：
                                                                                                                                                                                          
  1. 调整 Top-K 和相似度阈值（最简单）                                                                                                                                                    
                                                                                                                                                                                          
  // top_k: 返回几条结果                                                                                                                                                                  
  // score_threshold: 最低相似度，低于这个的不要                                                                                                                                          
  const results = await qdrant.search({                                                                                                                                                   
    vector: queryVector,                                                                                                                                                                  
    limit: 5,                // top_k = 5                                                                                                                                                 
    score_threshold: 0.7,    // 相似度低于 0.7 的过滤掉                                                                                                                                   
  });                                                                                                                                                                                     
                                                                                                                                                                                          
  - top_k 太大 → 塞了太多不相关内容，干扰 AI                                                                                                                                              
  - top_k 太小 → 可能漏掉重要信息                                                                                                                                                         
  - 一般 3~5 是甜点                                                                                                                                                                       
                                                                                                                                                                                          
  2. Metadata 过滤（简单有效）                                                                                                                                                            
                                                                                                                                                                                          
  每个 chunk 存入时带上 metadata，检索时可以先过滤：                                                                                                                                      
                                                        
  // 只在"产品需求"相关的文档中搜索，不搜技术文档                                                                                                                                         
  const results = await qdrant.search({                                                                                                                                                   
    vector: queryVector,                                                                                                                                                                  
    filter: {                                                                                                                                                                             
      must: [{ key: 'category', match: { value: '产品需求' } }]                                                                                                                           
    },                                                                                                                                                                                    
    limit: 5,                                                                                                                                                                             
  });                                                                                                                                                                                     
                                                                                                                                                                                          
  这样可以按文档类型、日期、作者等维度缩小搜索范围。                                                                                                                                      
                                                        
  3. 混合检索（Hybrid Search）                                                                                                                                                            
                                                        
  向量搜索擅长语义匹配（"登录" 能匹配到 "用户认证"），但不擅长精确关键词匹配（搜 "ERR_001" 这种编号）。                                                                                   
                                                        
  混合检索 = 向量检索 + 关键词检索（BM25），取两者交集或加权合并：                                                                                                                        
                                                        
  查询: "ERR_001 错误处理"                                                                                                                                                                
                                                                                                                                                                                          
  向量检索结果:                    关键词检索结果:                                                                                                                                        
  1. "错误处理最佳实践" (0.89)    1. "ERR_001 连接超时" (高)                                                                                                                              
  2. "异常捕获机制" (0.85)        2. "ERR_001 重试策略" (中)                                                                                                                              
  3. "日志记录规范" (0.82)        3. "ERR_002 参数错误" (低)                                                                                                                              
                                                                                                                                                                                          
  合并后:                                                                                                                                                                                 
  1. "ERR_001 连接超时"  ← 两边都有，排第一                                                                                                                                               
  2. "ERR_001 重试策略"                                                                                                                                                                   
  3. "错误处理最佳实践"                                                                                                                                                                   
                                                                                                                                                                                          
  Qdrant 原生支持混合检索。                                                                                                                                                               
                                                                                                                                                                                          
  4. Rerank（重排序）                                                                                                                                                                     
                                                        
  向量检索拿到 top 20 条结果后，再用一个更精确的模型对这 20 条重新打分排序，取 top 5：                                                                                                    
                                                        
  向量检索 (快，粗糙) → top 20 候选                                                                                                                                                       
        ↓                                                                                                                                                                                 
  Rerank 模型 (慢，精确) → top 5 最终结果                                                                                                                                                 
                                                                                                                                                                                          
  相当于"初筛 + 精选"。Rerank 模型会同时看查询和文档内容，比单纯向量距离更准。                                                                                                            
                                                                                                                                                                                          
  常用 Rerank 服务：                                                                                                                                                                      
  - Cohere Rerank                                       
  - 智谱 Rerank                                                                                                                                                                           
  - 本地 BGE-Reranker                                   
                                                                                                                                                                                          
  5. Query 改写                                                                                                                                                                           
                                                                                                                                                                                          
  用户的查询可能很模糊，先让 AI 改写成更适合检索的形式：                                                                                                                                  
                                                                                                                                                                                          
  原始查询: "登录有 bug"                                                                                                                                                                  
      ↓ AI 改写                                                                                                                                                                           
  改写后: "用户登录功能异常 身份验证失败 登录流程错误"                                                                                                                                    
                                                                                                                                                                                          
  或者拆成多个子查询，分别检索后合并结果：                                                                                                                                                
                                                                                                                                                                                          
  原始: "对比 React 和 Vue 的状态管理"                                                                                                                                                    
      ↓                                                                                                                                                                                   
  子查询1: "React 状态管理方案 Redux Zustand"                                                                                                                                             
  子查询2: "Vue 状态管理方案 Pinia Vuex"                                                                                                                                                  
                                                                                                                                                                                          
  6. Parent-Child 切片                                                                                                                                                                    
                                                                                                                                                                                          
  把文档切成小片段做检索（精准），但返回时给 AI 看大片段（完整上下文）：                                                                                                                  
                                                        
  大片段（Parent）: 整个小节，2000字符                                                                                                                                                    
    ├── 小片段1（Child）: 200字符  ← 用这个做向量检索                                                                                                                                     
    ├── 小片段2（Child）: 200字符                                                                                                                                                         
    └── 小片段3（Child）: 200字符                                                                                                                                                         
                                                                                                                                                                                          
  检索时: 用小片段匹配（精准）                          
  返回时: 找到小片段3匹配 → 返回它的 Parent（完整上下文）

  六、RAG 效果评估                                                                                                                                                                        
                                                                                                                                                                                          
  怎么知道 RAG 做得好不好？
                                                                                                                                                                                          
  关键指标                                              

  ┌────────────┬───────────────────────────────────┬─────────────────────────────────────────┐
  │    指标    │               含义                │                 怎么看                  │
  ├────────────┼───────────────────────────────────┼─────────────────────────────────────────┤
  │ 检索召回率 │ 相关的文档有没有被检索到          │ 手动检查 top K 里是否包含正确答案的来源 │
  ├────────────┼───────────────────────────────────┼─────────────────────────────────────────┤
  │ 检索精确率 │ 检索到的文档里有多少是真的相关的  │ top 5 里有 3 个相关 → 精确率 60%        │                                                                                            
  ├────────────┼───────────────────────────────────┼─────────────────────────────────────────┤                                                                                            
  │ 生成准确性 │ AI 有没有正确引用检索到的内容     │ AI 的回答是否与参考资料一致             │                                                                                            
  ├────────────┼───────────────────────────────────┼─────────────────────────────────────────┤                                                                                            
  │ 幻觉率     │ AI 有没有编造参考资料里没有的内容 │ 对比 AI 输出和原始文档                  │
  └────────────┴───────────────────────────────────┴─────────────────────────────────────────┘                                                                                            
                                                        
  简单的测试方法                                                                                                                                                                          
                                                        
  准备 10~20 个测试问题，每个问题你知道答案在哪个文档里：                                                                                                                                 
                                                        
  问题: "我们的密码策略要求是什么？"                                                                                                                                                      
  预期来源: 《安全规范》第4章                                                                                                                                                             
  预期答案: "密码至少8位，包含大小写字母和数字"                                                                                                                                           
                                                                                                                                                                                          
  实际检索结果:                                                                                                                                                                           
    1. 《安全规范》第4章 密码策略 ✅                                                                                                                                                      
    2. 《安全规范》第3章 账号管理 ✅ 相关                                                                                                                                                 
    3. 《员工手册》WiFi密码 ❌ 不相关                                                                                                                                                     
                                                                                                                                                                                          
  AI 输出: "根据安全规范，密码要求至少8位..." ✅ 正确

  七、你的项目怎么落地                                  
                                                                                                                                                                                          
  对应到你的工作流项目，RAG 的价值是：
                                                                                                                                                                                          
  让每个节点都能"读"你上传的文档，基于真实资料来执行任务，而不是 AI 凭空编造。                                                                                                            
                                                                                                                                                                                          
  比如一个软件开发工作流：                                                                                                                                                                
  - 需求分析节点 → 读《产品需求文档》                   
  - 架构设计节点 → 读《技术规范》《系统架构图》                                                                                                                                           
  - 代码实现节点 → 读《编码规范》《API 文档》           
  - 测试节点 → 读《测试用例模板》《质量标准》                                                                                                                                             
                                                                                                                                                                                          
  每个节点绑定不同的知识库，各取所需。


  针对你的项目（AI 工作流平台，节点执行时调用 RAG），你需要掌握的概念和会遇到的坑：                                                                                                       
                                                                                                                                                                                        
  ---                                                                                                                                                                                     
  你必须理解的概念（其他的可以先不管）
                                                                                                                                                                                          
  1. Embedding 调用                                     
                                                                                                                                                                                          
  你不需要理解向量怎么算的，只需要知道：                                                                                                                                                  
  - 它是一个 API 调用，输入文本，返回数字数组                                                                                                                                             
  - 入库时和检索时必须用同一个 Embedding 模型，否则向量空间不一致，搜不到东西                                                                                                             
  - 换模型 = 所有文档重新 Embedding 入库                                     
                                                                                                                                                                                          
  2. Chunk 大小和重叠                                                                                                                                                                     
                                                                                                                                                                                          
  这是你唯一需要反复调的参数：                                                                                                                                                            
  - chunkSize：每个片段多长                                                                                                                                                               
  - chunkOverlap：相邻片段重叠多少                                                                                                                                                        
                                                        
  3. Top-K 和 Score Threshold                                                                                                                                                             
                                                                                                                                                                                          
  检索时返回几条结果、最低相似度多少。这两个参数直接决定塞给 AI 的内容质量。                                                                                                              
                                                                                                                                                                                          
  4. Collection（知识库 = Collection）                                                                                                                                                    
                                                        
  Qdrant 里一个 Collection 就是一个知识库，里面存着所有文档的向量。你的项目里一个知识库对应一个 Collection。                                                                              
                                                        
  以上 4 个就够了，HNSW、余弦相似度这些底层原理不影响你开发。                                                                                                                             
                                                        
  ---                                                                                                                                                                                     
  你会遇到的坑（按优先级排序）                          
                                                                                                                                                                                          
  坑 1：检索到了但不相关（最常见）
                                                                                                                                                                                          
  表现：AI 的回答看起来有理有据，但引用的内容和问题无关。                                                                                                                                 
                                                                                                                                                                                          
  原因：chunk 切得不好，一个片段里混了多个主题。                                                                                                                                          
                                                        
  ❌ 坏的切片：                                                                                                                                                                           
  "第三章 用户认证。3.1 登录流程：用户输入账号密码...                                                                                                                                     
   3.2 权限管理：管理员可以分配角色...                                                                                                                                                    
   3.3 日志审计：所有操作都会记录..."                                                                                                                                                     
                                                                                                                                                                                          
  → 这个 chunk 包含了认证、权限、日志三个主题                                                                                                                                             
  → 搜"日志审计"时，这个 chunk 也会匹配到"登录流程"，AI 就可能答非所问                                                                                                                    
                                                                                                                                                                                          
  ✅ 好的切片：                                                                                                                                                                           
  chunk1: "3.1 登录流程：用户输入账号密码..."                                                                                                                                             
  chunk2: "3.2 权限管理：管理员可以分配角色..."                                                                                                                                           
  chunk3: "3.3 日志审计：所有操作都会记录..."                                                                                                                                             
                                                                                                                                                                                          
  解决：                                                                                                                                                                                  
  - 用递归切片（RecursiveCharacterTextSplitter），优先在段落边界切                                                                                                                        
  - chunk 不要太大，800 字符起步                                                                                                                                                          
  - 存 metadata 时记录标题/章节信息                     
                                                                                                                                                                                          
  坑 2：检索不到（明明文档里有）                                                                                                                                                          
                                                                                                                                                                                          
  表现：你确定文档里写了相关内容，但 top 5 里就是没有。                                                                                                                                   
                                                                                                                                                                                          
  原因和解决：                                                                                                                                                                            
                                                        
  ┌──────────────────────┬──────────────────────────────────────────┬──────────────────────────────────────────┐                                                                          
  │         原因         │                   例子                   │                   解决                   │
  ├──────────────────────┼──────────────────────────────────────────┼──────────────────────────────────────────┤                                                                          
  │ 用词不同             │ 查询说"报错"，文档写"异常处理"           │ 用 Query 改写，或混合检索（关键词+向量） │
  ├──────────────────────┼──────────────────────────────────────────┼──────────────────────────────────────────┤                                                                          
  │ chunk 太小           │ 关键信息被切到两个片段里，各自不完整     │ 增大 chunkOverlap                        │                                                                          
  ├──────────────────────┼──────────────────────────────────────────┼──────────────────────────────────────────┤                                                                          
  │ score_threshold 太高 │ 阈值 0.9，相似度 0.85 的相关内容被过滤了 │ 降低阈值到 0.6~0.7                       │                                                                          
  ├──────────────────────┼──────────────────────────────────────────┼──────────────────────────────────────────┤                                                                          
  │ top_k 太小           │ 相关内容排在第 6 名，但只取 top 5        │ 增大 top_k                               │
  └──────────────────────┴──────────────────────────────────────────┴──────────────────────────────────────────┘                                                                          
                                                        
  调试方法：最有效的办法是打印检索结果和分数：                                                                                                                                            
                                                        
  // 开发阶段加这个日志                                                                                                                                                                   
  console.log('[RAG] 查询:', query);                                                                                                                                                      
  results.forEach((r, i) => {                                                                                                                                                             
    console.log(`[RAG] #${i+1} score=${r.score.toFixed(3)} | ${r.content.substring(0, 80)}...`);                                                                                          
  });                                                                                                                                                                                     
                                                                                                                                                                                          
  看到分数和内容就能判断是切片问题还是检索参数问题。                                                                                                                                      
                                                        
  坑 3：AI 不用检索到的内容（给了资料但 AI 忽略）                                                                                                                                         
                                                        
  表现：检索结果明明包含正确答案，但 AI 用自己的知识回答了，甚至编造了内容。                                                                                                              
                                                        
  原因：prompt 写得不够强硬，AI 的"自信"压过了参考资料。                                                                                                                                  
                                                        
  ❌ 弱 prompt：                                                                                                                                                                          
  "以下是一些参考资料：{context}，请回答问题。"                                                                                                                                           
  → AI 可能觉得自己知道更好的答案，忽略资料                                                                                                                                               
                                                                                                                                                                                          
  ✅ 强 prompt：                                                                                                                                                                          
  "请严格基于以下参考资料回答问题。如果参考资料中没有相关信息，请明确说'资料中未提及'，不要编造。                                                                                         
                                                                                                                                                                                          
  参考资料：                                                                                                                                                                              
  {context}                                                                                                                                                                               
                                                                                                                                                                                          
  问题：{query}"                                                                                                                                                                          
   
  解决：在 system prompt 里明确约束 AI 必须基于资料回答。                                                                                                                                 
                                                        
  坑 4：重复内容干扰                                                                                                                                                                      
                                                        
  表现：检索到的 5 条结果里有 3 条说的是同一件事（因为重叠切片），浪费了 top_k 的名额。                                                                                                   
                                                        
  原因：chunkOverlap 太大，或者文档本身有重复内容。                                                                                                                                       
                                                        
  解决：                                                                                                                                                                                  
  - 检索后做去重：如果两条结果相似度 > 0.95，只保留分数高的
  - 适当减小 overlap（但不能完全去掉）                                                                                                                                                    
                                                        
  // 简单去重逻辑                                                                                                                                                                         
  const deduplicated = results.filter((r, i) => {                                                                                                                                         
    // 跟前面的结果对比，内容重叠超过 80% 就去掉                                                                                                                                          
    return !results.slice(0, i).some(prev =>                                                                                                                                              
      similarity(prev.content, r.content) > 0.8                                                                                                                                           
    );                                                                                                                                                                                    
  });                                                                                                                                                                                     
                                                                                                                                                                                          
  坑 5：文档更新后忘记重新入库                                                                                                                                                            
   
  表现：文档更新了，但 RAG 还在引用旧版内容。                                                                                                                                             
                                                        
  原因：向量数据库里存的还是旧文档的切片。                                                                                                                                                
                                                        
  解决：                                                                                                                                                                                  
  - 更新文档时，先删除该文档的所有旧切片，再重新切片入库
  - 每个 chunk 的 metadata 里记录 documentId，方便按文档删除                                                                                                                              
                                                            
  // 更新文档流程                                                                                                                                                                         
  await qdrant.delete({ filter: { documentId: docId } });  // 删旧的                                                                                                                      
  const newChunks = await splitDocument(newContent);         // 重新切片                                                                                                                  
  await qdrant.upsert(newChunks);                           // 入新的                                                                                                                     
                                                                                                                                                                                          
  坑 6：tokens 超限                                                                                                                                                                       
                                                                                                                                                                                          
  表现：检索到的内容太多，加上 prompt + history，超过了 AI 模型的上下文窗口。                                                                                                             
                                                        
  原因：top_k 太大，或者每个 chunk 太长。                                                                                                                                                 
                                                        
  解决：                                                                                                                                                                                  
  - 计算总 tokens：system prompt + history + RAG 内容 + 预留输出空间 < 模型上限
  - 一般给 RAG 内容预留 2000~4000 tokens（约 3000~6000 中文字符）                                                                                                                         
  - 如果 5 条结果超出预算，从末尾开始砍                          
                                                                                                                                                                                          
  // 控制 RAG 内容的总长度                                                                                                                                                                
  const MAX_RAG_CHARS = 4000;                                                                                                                                                             
  let totalChars = 0;                                                                                                                                                                     
  const selected = [];                                                                                                                                                                    
  for (const chunk of results) {                                                                                                                                                          
    if (totalChars + chunk.content.length > MAX_RAG_CHARS) break;                                                                                                                         
    selected.push(chunk);                                                                                                                                                                 
    totalChars += chunk.content.length;                                                                                                                                                   
  }                                                                                                                                                                                       
                                                        
  坑 7：Embedding API 批量调用太慢                                                                                                                                                        
                                                        
  表现：上传一个大文档，切出 200 个片段，逐个调 Embedding API 要好几分钟。                                                                                                                
                                                        
  解决：                                                                                                                                                                                  
  - 批量调用：大多数 Embedding API 支持一次传多个文本   
  - 并发限制：不要同时发 200 个请求，分批发（每批 20~50 个）                                                                                                                              
   
  // 批量 Embedding，每批 50 个                                                                                                                                                           
  const BATCH_SIZE = 50;                                                                                                                                                                  
  for (let i = 0; i < chunks.length; i += BATCH_SIZE) {                                                                                                                                   
    const batch = chunks.slice(i, i + BATCH_SIZE);                                                                                                                                        
    const embeddings = await embeddingApi.create({                                                                                                                                        
      input: batch.map(c => c.content),  // 一次传多个                                                                                                                                    
      model: 'embedding-3',                                                                                                                                                               
    });                                                                                                                                                                                   
    // 存入向量库...                                                                                                                                                                      
  }

  你的项目的调优路线                                    
                                                                                                                                                                                          
  分三个阶段，不要一上来就搞复杂的：
                                                                                                                                                                                          
  第一阶段：跑通（先做）                                                                                                                                                                  
                                                                                                                                                                                          
  固定长度切片（800字符，200重叠）                                                                                                                                                        
  + 单个 Embedding 模型                                                                                                                                                                   
  + 向量检索 top 5                                                                                                                                                                        
  + 硬 prompt 约束 AI 用资料回答                                                                                                                                                          
                                                                                                                                                                                          
  这就够用了，先验证流程通不通、效果够不够。                                                                                                                                              
                                                                                                                                                                                          
  第二阶段：调参（看效果再做）                                                                                                                                                            
                                                        
  + 加日志打印检索结果和分数                                                                                                                                                              
  + 调 chunkSize（试 500、800、1200 对比效果）                                                                                                                                            
  + 调 top_k（试 3、5、8）                                                                                                                                                                
  + 调 score_threshold（0.6~0.8）                                                                                                                                                         
  + 检索结果去重                                                                                                                                                                          
                                                                                                                                                                                          
  第三阶段：高级优化（有明确瓶颈再做）                                                                                                                                                    
                                                                                                                                                                                          
  + 混合检索（向量 + 关键词）                                                                                                                                                             
  + Rerank 重排序                                                                                                                                                                         
  + Query 改写                                                                                                                                                                            
  + Parent-Child 切片
