 一、认证与安全                                                                                                                                                                          
                                                                                                                                                                                          
  Q1: JWT 认证流程是怎么做的？                                                                                                                                                            
                                                                                                                                                                                          
  你要能说清完整链路：                                                                                                                                                                    
  - 注册：bcryptjs 加盐 hash（salt rounds=10）→ 存入 users 表                                                                                                                             
  - 登录：bcrypt.compare() 验证 → 签发 JWT（payload: { sub, email, role }）                                                                                                               
  - 前端：token 存 localStorage，axios 拦截器自动附 Authorization: Bearer  
  - 后端：JwtAuthGuard + passport-jwt 的 JwtStrategy 解析 token → req.user                                                                                                                
                                                                                                                                                                                          
  Q2: 有什么安全隐患？（面试官最爱追的）                                                                                                                                                  
                                                                                                                                                                                          
  你应该主动提到这些（说明你有思考）：                                                                                                                                                    
  - JWT Secret 降级：jwt.strategy.ts 里 fallback 到 'default_secret'，如果 .env 没配，攻击者可以伪造任意 token                                                                            
  - 无 token 刷新机制：签发后无法吊销，也没有 refresh token                                                                                                                               
  - 无登录限频：没有 rate limiting，可以暴力破解           
  - role 字段没用到：JWT 里带了 role 但没有任何 RBAC 守卫                                                                                                                                 
  - 前端 token 双源：axios 拦截器读 localStorage，AuthRoute 读 Zustand store，可能不一致                                                                                                  
                                                                                                                                                                                          
  Q3: 为什么用 bcryptjs 而不是 bcrypt？                                                                                                                                                   
                                                                                                                                                                                          
  bcryptjs 是纯 JS 实现，免去原生编译依赖，牺牲性能换可移植性。面试时说清这个 trade-off 就行。

  二、工作流执行引擎（最核心，最容易深问）              
                                                                                                                                                                                          
  Q4: 工作流是怎么按顺序执行的？
                                                                                                                                                                                          
  这是简化版 Kahn 算法：                                                                                                                                                                  
  1. 遍历所有 edges 计算每个节点的入度                                                                                                                                                    
  2. 找到入度为 0 的节点作为起点                                                                                                                                                          
  3. 用 nextMap（source → target）沿着连线逐个执行      
                                                                                                                                                                                          
  你必须能说出局限性：                                                                                                                                                                    
  - 只支持线性链，不支持 DAG（并行分支会丢失）                                                                                                                                            
  - nextMap.set(e.source, e.target) 一个节点多条出边只保留最后一条                                                                                                                        
  - find 找入度 0 的节点，多个起点只取第一个                                                                                                                                              
                                                                                                                                                                                          
  Q5: 节点间怎么传数据的？                                                                                                                                                                
                                                                                                                                                                                          
  getNodeInput() 通过画布的 edges 找到上游节点 → 读取上游 node_executions.output。面试官可能追问：如果一个节点有多个输入怎么办？— 当前用 edges.find() 只取一条，多输入不支持。            
                                                                                                                                                                                          
  Q6: 审批后怎么继续执行后续节点？                                                                                                                                                        
                                                        
  approveNode() 最后调 this.continueWorkflow(...).catch(() => {}) —— fire-and-forget 异步执行。面试官会问：如果后续节点执行失败呢？答：catch(() => {})                                    
  吞掉了错误，用户不会收到通知，这是已知的不足。        
                                                                                                                                                                                          
  Q7: executeNode 里的 upsert 是怎么回事？                                                                                                                                                
  
  避免并发执行同一节点时的唯一约束冲突。用 TypeORM 的 upsert，composite key 是 ['workflowId', 'nodeKey']。如果记录已存在就更新，不存在就插入。

  三、AI 集成                                           
                                                                                                                                                                                          
  Q8: 代码生成是怎么从 AI 输出变成文件的？              
                                                                                                                                                                                          
  callAI(prompt, maxTokens=16384)                                                                                                                                                         
    → AI 返回 Markdown（### `filepath` + ```code```)                                                                                                                                      
    → parseCodeBlocks() 正则解析                                                                                                                                                          
    → fs.mkdirSync + fs.writeFileSync 写入 AI_WORKSPACE_DIR/{workflowId}/                                                                                                                 
                                                                                                                                                                                          
  Q9: 为什么用两层正则？                                                                                                                                                                  
                                                                                                                                                                                          
  不同 AI 模型（DeepSeek、Claude、GPT）对格式遵守程度不同。严格正则匹配标准格式 ### \path`，如果匹配到了直接返回；如果匹配不到，用宽松正则兜底（兼容                                      
  ##、无反引号、**加粗**等偏差）。宽松正则额外加了/[./]/.test(filePath)` 过滤掉中文标题。
                                                                                                                                                                                          
  面试追问：宽松正则有什么风险？— [\s\S]*? 在标题和代码块之间可能跨越多个代码块误匹配。                                                                                                   
  
  Q10: 为什么 callAI 要加 maxTokens 参数？                                                                                                                                                
                                                        
  代码生成输出很长，默认 4096 容易截断。截断后最后一个代码块缺少闭合的 ```，正则匹配不到，那个文件就丢失了。所以代码生成传 16384。                                                        
                                                        
  Q11: 三层 Prompt 系统是怎么设计的？                                                                                                                                                     
                                                        
  system（系统内置默认）→ project（项目级覆盖）→ node（节点级覆盖）                                                                                                                       
  activeLayer: 1|2|3 决定用哪一层                                                                                                                                                         
  取值逻辑：取 activeLayer 对应的层，如果为 null 降级到 system 层，再 null 用硬编码默认值。这样既有开箱即用的默认 prompt，又允许逐层定制。

  四、SSE 流式对话                                                                                                                                                                        
                                                                                                                                                                                          
  Q12: 为什么不用 EventSource？                                                                                                                                                           
                                                                                                                                                                                          
  EventSource 只支持 GET，这里需要 POST（发消息体）+ 自定义 Authorization header。所以用 fetch() + ReadableStream 手动解析 SSE。                                                          
                                                                                                                                                                                          
  Q13: 后端 SSE 是怎么实现的？                                                                                                                                                            
                                                        
  手动设置三个 header（text/event-stream、no-cache、keep-alive），用 @Res() 注入原始 response，for await 遍历 OpenAI SDK 的 stream，每个 token 写 data:                                   
  ${JSON.stringify({text})}\n\n，最后发 data: [DONE]\n\n。
                                                                                                                                                                                          
  面试追问：为什么不用 NestJS 内置的 @Sse() 装饰器？— 因为它基于 Observable，包装 async iterator 反而更复杂。                                                                             
  
  Q14: 前端消费 SSE 流有什么问题？                                                                                                                                                        
                                                        
  - 网络 chunk 可能把一条 SSE 数据截断，split('\n') + JSON.parse 可能解析失败                                                                                                             
  - 每个 token 都触发 setMessages 创建新数组，大量 token 时性能有压力
  - 缺少 error 事件处理

  五、前端架构                                                                                                                                                                            
                                                                                                                                                                                          
  Q15: ReactFlow 拖拽是怎么实现的？                     
                                                                                                                                                                                          
  HTML5 Drag API：                                                                                                                                                                        
  - NodePanel 的 onDragStart 设 dataTransfer.setData('application/reactflow', JSON.stringify(nodeData))                                                                                   
  - NodePages 的 onDrop 读取数据，screenToFlowPosition() 转坐标，创建新 Node                                                                                                              
                                                                            
  Q16: 撤销/重做怎么做的？                                                                                                                                                                
                                                                                                                                                                                          
  useState 维护 history 数组 + historyIndex。useEffect 监听 [nodes, edges] 变化，300ms 防抖后 push 快照。undo/redo 时设 isUndoRedo.current = true 防止重复记录。                          
                                                                                                                                                                                          
  面试追问：有什么问题？— 每次拖拽节点位置都会产生快照，内存可能膨胀；useEffect 读了 historyIndex 但没放依赖数组，有闭包陈旧风险。                                                        
                                                        
  Q17: 执行状态怎么同步到画布上的？                                                                                                                                                       
                                                        
  3 秒轮询 getExecutions()，JSON 快照比较检测变化，更新节点 status。10 次无变化且无 running 节点 → 自动停止。StageNode 审批后通过 CustomEvent 通知 NodePages 重启轮询。                   
                                                        
  面试追问：为什么不用 WebSocket？— 当前规模轮询够用，WebSocket 需要维护连接状态和重连逻辑，投入产出不划算。                                                                              
                                                        
  Q18: 为什么用 CustomEvent 做组件通信？                                                                                                                                                  
                                                        
  StageNode 在 ReactFlow 的渲染树深处，和 NodePages 没有直接的 props 传递路径。CustomEvent 简单直接，但不类型安全、不可追踪。更好的方案是 Zustand store 或 React Context。

  六、安全防护                                          
                                                                                                                                                                                          
  Q19: 文件读取接口怎么防路径穿越？
                                                                                                                                                                                          
  path.resolve(fullPath).startsWith(path.resolve(workDir))
                                                                                                                                                                                          
  两边都用 path.resolve() 归一化（消除 ..），然后 startsWith 检查。比如 ../../etc/passwd → resolve 后变成 /etc/passwd，不以 workDir 开头，拒绝。                                          
                                                                                                                                                                                          
  面试追问：startsWith 有什么边界情况？— 如果 workDir 是 /tmp/abc，恶意路径指向 /tmp/abcdef/x 也会通过。但因为 workflowId 是 UUID，这种碰撞概率极低。更严格的做法是 workDir 末尾加 /。

  七、数据库设计                                        
                                                                                                                                                                                          
  Q20: 为什么 canvasData 存 JSON 而不是拆成关系表？     
                                                                                                                                                                                          
  ReactFlow 的 nodes/edges 结构复杂（位置、样式、自定义 data），拆成关系表会有大量 join 和序列化开销。JSON 列一次读写整个画布，适合"整体加载、整体保存"的场景。缺点是不能对画布内容做 SQL 
  查询。                                                                                                                                                                                  
                                                                                                                                                                                          
  Q21: synchronize: true 有什么风险？                                                                                                                                                     
  
  TypeORM 自动根据 Entity 修改数据库 schema。开发方便，但生产环境可能丢数据（比如删了一个 Column，对应列直接 DROP）。生产应该用 migration。

  面试不只是被动回答，你应该主动讲这些：

  4. fire-and-forget 审批续跑：用户不用等后续节点执行完才收到审批响应，体验更好                                                                                                           
  5. 路径穿越防护：path.resolve + startsWith 标准做法
