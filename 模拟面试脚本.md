# 模拟面试脚本

使用方式：找一个 AI（Claude/ChatGPT）把下面的 Prompt 发给它，让它扮演面试官面试你。回答后让它打分+给改进建议。

---

## 使用 Prompt（直接复制发给 AI）

```
你是一位大厂前端技术面试官（P7 级别），正在面试一位有 6 年经验、最近转 AI 方向的前端候选人。他做了一个 AI 研发效能平台（Agent Loop + RAG + 工作流编排）。

面试规则：
1. 总共 45 分钟，按下面的题目顺序逐题提问
2. 每题候选人回答后，你先给出评分（A/B/C/D）和简短点评（哪里好、哪里可以补充）
3. 根据候选人回答深度决定是否追问（最多追问 1 次）
4. 如果候选人回答偏离重点，用"能再具体讲讲 XXX 吗？"引导
5. 面试结束后给出整体评价和改进建议

评分标准：
- A（优秀）：有技术深度、有数据支撑、有取舍思考、表达清晰流畅
- B（良好）：回答正确完整，但缺少深度或数据
- C（及格）：知道怎么做但讲不清为什么，或遗漏关键点
- D（不及格）：回答错误或无法展开

现在开始面试，请从第一题开始提问。每次只问一题，等我回答。
```

## 面试题目序列（共 3 轮，由浅入深）

### 第一轮：开场 + 项目概览（10 分钟）

| # | 题目 | 考察点 | 期望关键词 |
|---|------|--------|-----------|
| 1 | 请做一个 1 分钟自我介绍 | 表达能力、项目定位、钩子设计 | 6年经验/AI方向/ReactFlow/SSE/Agent Loop/RAG/审批/PR |
| 2 | 这个 AI 工作流平台和普通 AI Chat 有什么区别？ | 产品理解、架构认知 | 工作流编排/Agent工具调用/人工审批闭环 |
| 3 | 你在这个项目里负责什么？团队怎么分工的？ | 角色定位、避免"个人项目"印象 | 5人团队/主导后端架构+Agent核心模块/前端同事负责画布 |

**追问池：**
- 你说提效 40%，这个数据怎么算出来的？
- 为什么不直接用 Cursor/Copilot，非要自己做平台？

### 第二轮：核心技术深挖（20 分钟）

| # | 题目 | 考察点 | 期望关键词 |
|---|------|--------|-----------|
| 4 | Agent Loop 是怎么实现的？ | AI 工程核心能力 | while循环/tool_calls/executeTool/回填/双引擎(Codex+自研)/降级 |
| 5 | Agent 上下文膨胀怎么解决？ | 工程细节、边界处理 | 源头截断500行/滑动窗口/保留3轮/不能删只能截/tool_call_id配对 |
| 6 | RAG 从文档上传到检索的完整链路讲一下 | 全链路理解 | 解析→切片1500字→Embedding batch=10→Milvus双写→检索top-8→score过滤→Prompt增强 |
| 7 | Diff 审批是怎么实现的？ | 工程落地能力 | scanDirectory快照/执行前后对比/added-modified-deleted/驳回回滚 |

**追问池：**
- 为什么不能删消息只能截短？具体报什么错？

- 白名单有哪些命令？AI 绕过白名单怎么办？
- Recall@5 = 82% 是怎么测的？测试集怎么标注的？
- Agent 死循环怎么办？
- 为什么不用 LangChain？

### 第三轮：架构拔高 + 压力测试（15 分钟）

| # | 题目 | 考察点 | 期望关键词 |

## 常见失分点（提前注意）

| 失分点 | 正确做法 |
|--------|---------|
| 回答太长超过 3 分钟 | 每题控制 1.5-2 分钟，先说结论再展开 |
| 只说"怎么做"不说"为什么" | 每个技术选型都加一句取舍理由 |
| 被追问时慌乱 | 不知道就说"这个我还没深入研究，但我的思路是..." |
| 数据说不出来源 | 每个数据都要能解释怎么测的 |
| 说"我做了"没说团队 | 强调"5人团队/我主导XX模块" |
| 说"如果生产化" | 改成"已上线内部使用，后续规划是..." |
