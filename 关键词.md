# 上下文膨胀是怎么解决的？
因为 agent 循环的时候，会把 assistant 的内容 和 messages 数组一起发给 agent， 循环次数增加了以后，上下文的内容就会很庞大
所以我们通过几点去做限制：
第一点就是使用源头截断的方法，把模型 执行工具返回的内容：比如说 读文件、搜索代码等等这些操作返回的消息进行长度限制。
第二点就是使用滑动窗口方案，保留最近几次的完整上下文内容，把模型以前读文件、搜索代码等等的一些操作返回的内容进行一个压缩，然后把内容压缩成 
第几轮做了什么，调用了什么工具，返回了什么结果，这样的一句话
第三点就是使用截短内容的方式，因为 tool calling 规定 assistant 里的 tool_call_id 必须和后面的 tool result 进行 一一配对，如果不匹配会报错400.所以我们只是把content的内容进行了截短，本身的数据结构和 tool_call_id 的配对关系是保留的。
第四点就是使用token预算压缩，在每轮循环前都会评估当前上下文是否超过了 模型窗口的80%，如果超过了，就会把以前的消息进行一个压缩并且替换成摘要。
