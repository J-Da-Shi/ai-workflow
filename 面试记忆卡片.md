## 必背 6 题（每题 3-5 个关键词撑起回答）

### 1. Agent Loop 怎么实现？

```
关键词：while 循环 / tool_calls / executeTool 分发 / tool result 回填 / 双引擎（Codex优先，降级自研）
```

骨架：while 循环 → 发 messages 给模型 → 模型返回 tool_calls → 解析工具名+参数 → executeTool 执行 → 结果回填为 role=tool → 下一轮 → 直到无 tool_calls

追问"为什么不用 LangChain"：链路短（100行搞定）、需要精细控制（batch=10、重试退避）、依赖膨胀

---

### 2. 上下文爆炸怎么解决？

```
关键词：源头截断 500 行 / 滑动窗口 / 保留最近 3 轮 / 不能删只能截 / tool_call_id 配对
```

骨架：
- 第一层：源头截断 → read_file 超 500 行只返回 200 行 + 提示精确读取
- 第二层：滑动窗口 → messages>12 触发 → 保留最近 3 轮完整 → 早期 tool result 截断为 200 字符
- 关键约束：不能删消息！API 要求 tool_call 和 tool_result 通过 tool_call_id 配对

追问"Token 爆了"：极端情况保留轮数降到 1；更好方案是 Token 预算制（tiktoken 算总量）

---

### 3. RAG 链路？

```
关键词：解析 → 切片(1500字/200重叠) → Embedding(batch=10) → Milvus双写 → 检索top-8 → score>0.3 → Prompt增强
```

骨架：
- 入库：上传 → pdf-parse/mammoth 解析 → 递归字符切片 → 阿里云 text-embedding-v3 → 双写 Milvus+MySQL
- 检索：PRD 文本转向量 → Milvus search(filter+COSINE+top-8) → 过滤 score<0.3 → 拼进 System Prompt
- 数据：Recall@5=82%，审核准确率 61%→89%（30 份报告人工评判）

追问"不准怎么办"：Rerank 二次精排（A/B 测 91%）+ BM25 混合检索 + Query 改写

---

### 4. Diff 审批怎么生成？

```
关键词：scanDirectory / 执行前快照 / 执行后对比 / added-modified-deleted / react-diff-viewer / 驳回回滚
```

骨架：
- 执行前：scanDirectory 递归扫描存为 Map（排除 node_modules/.git/dist）
- 执行后：再扫描 → 对比得到 added/modified/deleted
- 驳回回滚：added 删除、modified 用旧内容覆盖、deleted 从快照恢复
- 为什么不用 git diff：不是所有节点都配了 Git

---

### 5. 安全怎么做？

```
关键词：safePath(path.resolve+startsWith) / run_command 白名单 / 60s 超时 / Git Token 不暴露
```

骨架：三道墙
- 文件层：safePath → resolve 解析真实路径 → 检查是否以 workDir 开头 → ../../etc/passwd 直接拒绝
- 命令层：白名单（npm run build/test/npx tsc）→ 为什么不用黑名单：rm 可以写成 \rm、$(rm...)
- Git 层：Token 存节点配置 → 执行引擎读取 → Agent 工具列表里没有 git 工具 → AI 看不到 Token

---

### 6. SSE 为什么不用 WebSocket？

```
关键词：单向推送 / HTTP 兼容 / 不用 EventSource(只支持GET) / fetch+getReader / 断连不影响后端
```

骨架：
- 单向推送场景，不需要客户端回传
- SSE 基于 HTTP，兼容鉴权/代理，不需要连接升级/心跳
- 不用 EventSource：只支持 GET 且不能带 Header → 我需要 POST+JWT → 用 fetch+ReadableStream 手动解析
- 断连兜底：SSE 断了后端照跑 + agent_logs 持久化 + 轮询兜底(3s 刷新)
