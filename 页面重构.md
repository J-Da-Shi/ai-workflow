前端 Drawer 页面重构计划                                                                                                                                                                    
                                                        
 Context

 当前右侧 Drawer 有 3 个平行 tab（节点配置 / AI 对话 / 执行日志），问题：
 - nodeConfig 塞了太多内容（基本信息 + Prompt + 输入输出 + 按钮），滚动很长
 - 执行后要手动切 tab 才能看日志，体验割裂
 - 三个 tab 关系不清晰

 参考：Cursor（对话+工具调用融合为一条流）+ Dify（配置和运行分两个模式）

 新设计

 将 3 tab 改为 2 tab：配置 + 运行

 ┌─────────── Drawer (820px) ───────────┐
 │ [icon] 节点名称  [状态]               │
 │  [配置]  [运行]                       │
 ├───────────────────────────────────────┤
 │                                       │
 │ 配置 tab:                             │
 │   Prompt 编辑器（三层 radio）          │
 │   高级设置（折叠：模型/超时/审批）     │
 │                                       │

 │     thinking → tool_done → done       │
 │   底部操作栏：执行 / 通过 / 驳回      │
 │                                       │
 └───────────────────────────────────────┘

 配置 tab 精简

 保留：
 - Prompt 三层编辑器（核心交互，占主要空间）
 - 高级设置（Collapse 折叠，默认收起）：模型、超时、审批开关、输入来源

 移除：
 - 基本信息 section → 节点类型在 header 已体现
 - 执行数据（Token/耗时）→ 移到运行 tab
 - 输入/输出数据 → 移到运行 tab
 - 执行按钮 → 移到运行 tab 底部
