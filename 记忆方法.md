 一、记忆方法：三层记忆法

  第 1 层：背项目主线

  先记住这个项目的总叙事：

  + Diff 审批回滚
  + Git PR 自动化

  你每次回答都尽量往这条主线靠。

  第 3 层：背标准回答模板

  每个问题都按这个模板答：

  1. 背景：为什么需要这个设计？
  2. 实现：我怎么做的？
  3. 难点：遇到什么工程问题？
  4. 取舍：为什么不用其他方案？
  5. 改进：生产化怎么优化？

  这个模板非常适合大厂面试。

  二、两周记忆计划
  
  第 1-2 天：压缩材料

  目标：不要直接背 总结面试.md，先把它压缩成卡片。
  
  你要整理出 4 类内容：

  1. 30 秒项目介绍
  2. 3 分钟项目介绍
  3. 20 个高频问题
  4. 每个问题 5 个关键词
  
  建议你先准备这几个核心问题：

  1. 你这个项目是做什么的？
  2. 整体架构怎么设计？
  3. Agent Loop 怎么实现？
  4. 为什么不用 LangChain？
  5. RAG 检索怎么做？
  6. RAG 效果怎么评估？
  7. 怎么防止 Agent 乱改文件？
  8. Diff 审批怎么实现？
  9. 为什么用 SSE 不用 WebSocket？
  10. 工作流执行引擎怎么设计？
  11. 如果支持 DAG 并行怎么改？
  12. Git PR 自动化怎么做？
  13. 上下文爆炸怎么解决？
  14. 并发怎么处理？
  15. 生产化还有哪些问题？

  第 3-5 天：主攻 Agent Loop

  这部分最值钱，必须背熟。

  每天练这几个问题：

  Agent Loop 是什么？
  你怎么实现 Agent Loop？
  和 Cursor / Claude Code 有什么区别？
  为什么不用 LangChain？
  上下文太长怎么办？
  工具调用怎么保证安全？
  run_command 为什么要白名单？

  你要能脱口而出这句话：

  ▎ Agent Loop 本质是一个 while 循环。每一轮把 system 
  ▎ prompt、用户任务和历史消息发给模型，模型如果返回 
  ▎ tool_calls，后端就解析工具名和参数，执行对应的文件工具，比如 
  ▎ read_file、search_code、write_file、run_command，再把工具结果作为 tool 
  ▎ message 回填给模型。这个循环一直执行到模型不再调用工具，返回最终总结。

  第 6-7 天：主攻 RAG

  重点背：

  解析 → 切片 → Embedding → Milvus 入库 → 检索 → Prompt 增强 → AI 审核

  你要能讲清楚：

  ▎ 我的 RAG 不是普通问答，而是接在 PRD 审核节点里。用户上传企业规范、历史评审标
  ▎ 准等文档后，系统会解析文档、切片、批量 Embedding，并把向量和元数据分别存到 
  ▎ Milvus 和 MySQL。审核 PRD 时，先把 PRD 文本转向量，在用户选择的知识库里做 
  ▎ topK 检索，再用 score threshold 过滤低相关片段，最后把命中的规范拼进 
  ▎ Prompt，让 AI 按企业标准输出审核报告。

  一定要准备一个“不足”：

  ▎ 当前还没有正式评测集，生产化会准备标注 query，统计 Recall@3、Recall@5、MRR 
  ▎ 和引用准确率。

  驳回怎么回滚？

  第 10 天：主攻前端表达

  前端不要讲太浅，要讲出工程点：

  React Flow 自定义节点
  拖拽节点
  画布保存
  撤销恢复
  状态轮询
  SSE 日志
  ReadableStream
  ReactDiffViewer
  CustomEvent

  SSE 这个回答要背熟：

  ▎ 我没有用原生 EventSource，因为 EventSource 只支持 GET，而且不好带 
  ▎ Authorization Header。节点执行需要 POST，并且要带 JWT，所以我用 fetch + 
  ▎ ReadableStream 手动读取响应流，按 data: 解析 SSE

  第 11-12 天：模拟面试

  不要再看文档背了，要开始闭卷说。

  方法：

  1. 随机抽一个问题。
  2. 不看文档，先说 90 秒。
  3. 录音。
  4. 回听。
  5. 看是否有这 5 点：

  有没有背景？
  有没有实现细节？
  有没有技术关键词？
  有没有取舍？
  有没有改进方向？

  如果没有，就补。

  第 13 天：压力追问

  专门练大厂喜欢追问的弱点：

  你这个项目生产能用吗？
  并发高了怎么办？
  RAG 不准怎么办？
  Agent 写错代码怎么办？
  Token 爆了怎么办？
  为什么不用成熟框架？
  Git token 泄露怎么办？
  怎么支持多分支 DAG？

  回答原则：

  ▎ 先承认边界，再说当前方案，最后说生产化改进。

  例如：

  ▎ 当前实现是功能闭环阶段，还不是完整生产级。比如 Agent 锁是进程内 
  ▎ Map，多实例部署时会失效。生产化我会把锁迁到 Redis，并引入任务队列限制 Agent 
  ▎ 并发，同时对 LLM API 做限流和重试。
  
  这种回答比硬吹更像真实项目。

  第 14 天：最终压缩

  最后一天只背一页纸。

  你的一页纸应该包含：

  项目一句话
  30 秒介绍
  5 个核心亮点
  10 个高频问题关键词
  5 个风险点
  5 个生产化改进

  不要再扩展新内容。

  三、每天怎么练

  每天建议 90 分钟。

  20 分钟：复习昨天的问题
  30 分钟：学习今天的新模块
  30 分钟：闭卷口述
  10 分钟：记录卡壳点

  最重要的是 闭卷口述。

  光看文档会有错觉：你以为会了，其实面试时说不出来。

  四、你要做一个“问题卡片”

  每张卡片长这样：

  Q：Agent Loop 怎么实现？

  关键词：
  while loop / tool_calls / executeTool / tool result / messages / SSE / context
   compress

  标准回答：
  1. 背景：普通 LLM 只能生成文本，不能操作项目文件。
  2. 实现：模型返回 tool_calls，后端执行工具，再把结果回填。
  3. 难点：多轮后上下文膨胀。
  4. 解决：read_file 源头截断 + compressMessages 滑动窗口。
  5. 改进：加入 edit_file，减少 write_file 全量输出 token。

  你不需要背完整句子，只要背关键词和顺序。
