# 两周面试背诵计划

目标：两周内把 `总结面试.md` 里的项目面试题记熟，达到面试官问到相关问题时，能在 5 秒内想起回答结构，60-90 秒内完整表达，并能补充技术取舍和生产化改进。

## 一、核心记忆原则

不要逐字硬背，要背「结构 + 关键词 + 口述顺序」。

每个问题都按下面 5 步回答：

```text
1. 背景：为什么需要这个设计？
2. 实现：我具体怎么做？
3. 难点：遇到什么工程问题？
4. 取舍：为什么不用其他方案？
5. 改进：生产化还能怎么优化？
```

项目总主线：

```text
AI 研发工作流平台
= React Flow 可视化编排
+ NestJS 工作流执行引擎
+ RAG 知识库审核 PRD
+ Agent Loop 自动写代码
+ SSE 实时执行日志
+ Diff 审批回滚
+ Git PR 自动化
```

记忆口诀：

```text
画布编排，节点执行；
RAG 审核，Agent 编码；
SSE 可视，Diff 审批；
Git 交付，安全兜底。
```

对应展开：

```text
画布编排：React Flow
节点执行：ExecutionService
RAG 审核：Milvus + Embedding
Agent 编码：Tool Calling Loop
SSE 可视：fetch + ReadableStream
Diff 审批：snapshot + rollback
Git 交付：branch + commit + PR
安全兜底：safePath + command whitelist
```

### 第 3-5 天：主攻 Agent Loop

这是项目最核心、最有面试价值的部分。

必须背熟的问题：

```text
1. Agent Loop 是什么？
2. 你怎么实现 Agent Loop？
3. 和 Cursor / Claude Code 有什么区别？
4. 为什么不用 LangChain？
5. 上下文太长怎么办？
6. 工具调用怎么保证安全？
7. run_command 为什么要白名单？
```

Agent Loop 标准回答骨架：

```text
Agent Loop 本质是一个 while 循环。
每一轮把 system prompt、用户任务和历史消息发给模型。
如果模型返回 tool_calls，后端就解析工具名和参数，执行对应工具。
比如 list_directory、read_file、search_code、write_file、run_command。
工具执行结果会作为 tool message 回填给模型。
这个循环一直执行到模型不再调用工具，返回最终总结。
```

关键词：

```text
while loop
tool_calls
executeTool
tool result 回填
messages
上下文压缩
safePath
run_command 白名单
SSE 日志
```

### 第 6-7 天：主攻 RAG

必须背熟这条链：

```text
文档上传
→ 文档解析
→ 文本切片
→ 批量 Embedding
→ Milvus 向量入库
→ MySQL 元数据存储
→ PRD 文本转向量
→ topK 检索
→ score threshold 过滤
→ Prompt 增强
→ AI 输出审核报告
```

RAG 标准回答骨架：

```text
我的 RAG 不是普通问答，而是接在 PRD 审核节点里。
用户上传企业规范、历史评审标准后，系统会解析文档、切片、批量 Embedding。
向量存到 Milvus，文档和 chunk 元数据存到 MySQL。
审核 PRD 时，先把 PRD 文本转向量，在用户选择的知识库里做 topK 检索。
再用 score threshold 过滤低相关片段，最后把命中的规范拼进 Prompt。
AI 基于这些企业标准输出结构化审核报告。
```

一定要主动准备不足：

```text
当前还没有正式评测集。
生产化会准备标注 query，统计 Recall@3、Recall@5、MRR、引用准确率和平均检索耗时。
必要时加 reranker 做二次排序。
```

### 第 8-9 天：主攻工作流、审批、Git

背熟这条链：

```text
React Flow 画布
→ nodes / edges 保存
→ ExecutionService 读取画布
→ 按连线执行节点
→ 节点进入 waiting
→ 人工 approve / reject
→ snapshot diff
→ Git commit
→ 审批后 push + PR
```

重点问题：

```text
1. 工作流怎么执行？
2. 节点之间怎么传数据？
3. 为什么要人工审批？
4. Diff 怎么生成？
5. 驳回怎么回滚？
6. Git token 安全怎么保证？
```

Diff 审批标准回答骨架：

```text
执行前，我会递归扫描工作目录，生成 snapshot，记录每个文件的路径和内容。
Agent 修改完成后，再扫描一次当前目录。
两次结果对比后得到 added、modified、deleted 三类变更。
前端用 ReactDiffViewer 展示 before / after。
用户审批通过则保留结果并继续后续节点。
如果驳回，就根据 snapshot 恢复 modified 和 deleted 文件，删除 added 文件。
```

### 第 10 天：主攻前端表达

前端不要只说页面，要讲工程点。

关键词：

```text
React Flow 自定义节点
拖拽节点
画布保存
撤销恢复
状态轮询
SSE 日志
fetch + ReadableStream
ReactDiffViewer
CustomEvent
```

SSE 标准回答骨架：

```text
我没有用原生 EventSource，因为 EventSource 只支持 GET，而且不好带 Authorization Header。
节点执行需要 POST，并且要带 JWT。
所以我用 fetch + ReadableStream 手动读取响应流。
后端返回 text/event-stream，前端用 reader 逐 chunk 读取，按 data: 解析 SSE 消息。
解析到 thinking、tool_done、done、error 事件后，实时更新执行日志。
```

### 第 11-12 天：模拟面试

开始闭卷说，不再只看文档。

训练方法：

```text
1. 随机抽一个问题。
2. 不看文档，口述 60-90 秒。
3. 录音。
4. 回听。
5. 检查有没有背景、实现、难点、取舍、改进。
6. 卡壳的问题重新做卡片。
```

每次回答都检查这 5 点：

```text
有没有解释为什么做？
有没有讲实现细节？
有没有说出技术关键词？
有没有讲方案取舍？
有没有补充改进方向？
```

### 第 13 天：压力追问

专门练大厂喜欢追问的不足和边界。

高频压力问题：

```text
1. 你这个项目生产能用吗？
2. 并发高了怎么办？
3. RAG 不准怎么办？
4. Agent 写错代码怎么办？
5. Token 爆了怎么办？
6. 为什么不用成熟框架？
7. Git token 泄露怎么办？
8. 怎么支持多分支 DAG？
9. 如果 LLM API 限流怎么办？
10. 如果 Agent 死循环怎么办？
```

回答原则：

```text
先承认边界，再说当前方案，最后说生产化改进。
```

示例：

```text
当前实现是功能闭环阶段，还不是完整生产级。
比如 Agent 锁是进程内 Map，多实例部署时会失效。
当前它能防止单进程内同一节点重复执行。
生产化我会把锁迁到 Redis，并引入任务队列限制 Agent 并发，同时对 LLM API 做限流和重试。
```

### 第 14 天：最终压缩

## 五、卡片模板

每道题按这个格式整理：

```text
Q：Agent Loop 怎么实现？

关键词：
while loop / tool_calls / executeTool / tool result / messages / SSE / context compress

标准回答：
1. 背景：普通 LLM 只能生成文本，不能操作项目文件。
2. 实现：模型返回 tool_calls，后端执行工具，再把结果回填。
3. 难点：多轮后上下文膨胀。
4. 解决：read_file 源头截断 + compressMessages 滑动窗口。
5. 改进：加入 edit_file，减少 write_file 全量输出 token。
```

## 六、30 秒项目介绍

```text
我做了一个 AI 驱动的软件开发工作流平台，定位是团队级 AI 研发流水线。
前端用 React Flow 做可视化节点编排，后端用 NestJS 实现执行引擎。
用户可以把 PRD 审核、需求评审、代码开发、自测、Code Review 等节点串起来执行。
普通节点走 LLM 生成，PRD 审核节点会走 RAG，从企业知识库检索规范后输出审核报告。
代码开发节点会启动自研 Agent Loop，通过 Tool Calling 自主读取文件、搜索代码、写文件和执行验证命令。
执行完成后系统会生成 Diff，只有人工审批通过后才会继续后续节点或创建 Git PR。
这个项目的核心不是简单调模型，而是把 Agent、RAG、审批、Git 和可观测日志串成一个工程化闭环。
```

## 七、面试表达注意事项

1. 不要把项目说成普通 Chatbot。
2. 不要只讲“调用 AI 接口”，要讲工程闭环。
3. 每个模块都要讲出为什么这么设计。
4. 主动承认不足，比硬吹更可信。
5. 遇到不会的问题，用“当前方案 + 生产化改进”回答。
6. 回答控制在 60-90 秒，不要无限展开。
7. 高频关键词要脱口而出：Agent Loop、Tool Calling、RAG、Milvus、SSE、Diff、审批、safePath、白名单、上下文压缩。

## 八、最终目标

两周后你要达到：

```text
1. 30 秒能讲清项目是什么。
2. 3 分钟能讲清项目架构和核心链路。
3. Agent Loop / RAG / Diff 审批 / SSE / 安全 5 个模块能深入讲。
4. 面对追问能说出当前不足和生产化优化。
5. 不依赖文档，能闭卷口述 20 个高频问题。
```
